# mentor-toolkit

정부 청년 멘토링 사업의 운영기관 제출 양식 자동화 **Claude Code** 플러그인입니다. 멘토가 Notion 연결과 기본 설정을 한 번만 해두면, 이후 멘티 자료를 건네주는 것만으로 등록·분석·회차 진행·PDF 보고서까지 한 흐름으로 처리됩니다.

---

## 처음 사용하는 멘토를 위한 가이드

### 1단계. 시작 전 준비 (1회)

세 가지를 미리 갖춰두면 됩니다.

**Claude Code CLI**  
https://docs.claude.com/en/docs/claude-code 에서 설치.

**Notion MCP 연결**  
`~/.claude.json` 또는 작업 디렉터리의 `.mcp.json`에 Notion MCP를 등록합니다. `claude mcp add notion ...` 명령이나 공식 가이드 참고.

**Python 의존성 (PDF 생성용)**  
처음 보고서를 만들 때 안내가 뜨지만, 미리 설치해두면 더 매끄럽습니다. **venv 사용을 권장합니다** (시스템 Python 오염 방지).

```bash
# 1. 전용 venv 생성 (한 번만)
python3 -m venv ~/.venvs/mentor-toolkit
source ~/.venvs/mentor-toolkit/bin/activate

# 2. 패키지 설치
pip install weasyprint jinja2 mplfonts requests
```

이후 Claude Code를 실행하기 전 같은 셸에서 `source ~/.venvs/mentor-toolkit/bin/activate`만 해두면 됩니다.

macOS라면 시스템 라이브러리도 필요합니다.
```bash
brew install pango
```

> venv를 쓰지 않을 거라면 `pip install --user weasyprint jinja2 mplfonts requests`를 권장합니다. 시스템 Python 보호 정책(PEP 668)을 무시하는 `--break-system-packages`는 마지막 수단으로만 사용하세요.

### 2단계. 플러그인 설치

```bash
# 마켓플레이스 등록
claude plugin marketplace add https://github.com/vljh246v/mentor-toolkit

# 플러그인 설치
claude plugin install mentor-toolkit@mentor-marketplace
```

설치 확인은 `claude plugin list`, 업데이트는 아래 순서로:

```bash
claude plugin marketplace update mentor-marketplace
claude plugin install mentor-toolkit@mentor-marketplace
```

### 3단계. 셋업 (1회만)

Claude Code를 열고 이렇게 입력하면 됩니다.

```
> 셋업해줘
```

`setup-mentor-toolkit`이 성명·소속·직위·분야·Notion 위치를 한 번에 묶어 물어봅니다. 답하면 다음이 자동으로 만들어집니다.

- `📚 멘토링 자동화 허브` 페이지
- `⚙️ 멘토링 설정` — 분야·정보 변경은 여기서
- `🧑‍💻 멘티 트래커` 데이터베이스
- `📝 템플릿 모음` 7장
- `🚀 사용 가이드`

### 4단계. 새 멘티 등록

멘티가 새로 생길 때마다 이력서나 자기소개서 파일 경로를 건네거나 이름과 고민을 직접 말해주면 됩니다. 두 번째, 세 번째 멘티도 똑같이 하면 됩니다.

```
> 김지훈 멘토링 시작. 이력서: ~/Downloads/김지훈_이력서.pdf
```

`init-mentoring`이 부족한 정보만 묶어서 물어본 뒤 등록·분석·다음 액션까지 한 번에 처리합니다. 이미 등록된 멘티 이름과 겹치면 새 사이클인지 기존 추가 회차인지 확인합니다.

분석이 끝나면 1차 시작 전에 운영기관 제출용 사전 평가지를 바로 만들 수 있습니다.

```
> 김지훈 사전 평가지 만들어줘
```

`pre-assessment`가 멘티 분석 내용을 바탕으로 **참여자역량결과보고서 PDF**를 생성합니다.

### 5단계. 회차가 끝나면 — end-session

회차가 끝나면 Notion 회차 페이지에 기록해두고 이렇게 말하면 됩니다.

```
> 김지훈 1차 끝
```

`end-session`이 Notion 회차 페이지를 먼저 읽습니다. 이미 써둔 내용은 건드리지 않고, 빠진 부분만 한 번에 물어봅니다. 트래커 회차 카운트도 자동으로 갱신됩니다.

### 6단계. 회차 사이에는 — between-sessions

회차 사이에 생기는 일들은 자연어로 말하면 됩니다.

| 멘토가 한 말 | 처리 |
|------------|------|
| `김지훈 자소서 v2 봐줘 ~/Downloads/v2.txt` | 검토 → 점수·코멘트 + 멘티에게 보낼 답장 초안 + 1차 페이지에 자동 기록 |
| `김지훈 인프런 강의 못 들었대` | 원인 진단 + 4가지 옵션(미루기/줄이기/대체/그만두기) + 추천 제시 |
| `김지훈 강의 절반 들었대` | 부분 진척 기록 → 다음 회차 어젠다에 후반 마무리 추가 |

### 7단계. 다음 회차 시작 — start-session

1차가 끝나고 2주 뒤 2차를 시작할 때, 멘티 컨텍스트를 다시 불러올 필요 없습니다.

```
> 김지훈 2차 시작
```

`start-session`이 이전 회차 노트·멘티 분석·로드맵을 Notion에서 한 번에 가져와 1분 브리핑 카드로 정리해줍니다. 새 회차 페이지도 직전 과제 점검 표와 어젠다가 미리 채워진 상태로 자동 생성됩니다.

### 8단계. 멘토링 도중 자연어로

| 멘토가 한 말 | 발동되는 스킬 |
|------------|------------|
| `김지훈 회사 추천해줘` | `company-recommender` |
| `김지훈 이력서 리뷰` | `resume-reviewer` |
| `김지훈 면접 질문 만들어줘` | `interview-prep` |
| `김지훈 모의 면접` | `mock-interviewer-agent` |
| `김지훈 분석 다시` | `mentee-analyzer` |

### 9단계. 멘토링 종료 — PDF 보고서

```
> 김지훈 보고서 PDF 만들어줘
```

`create-report`가 PDF 2개를 현재 작업 디렉터리에 직접 생성합니다.

- `김지훈_별지3-1_멘토링일지.pdf` — 회차별 합본
- `김지훈_별지3-2_결과보고서.pdf` — 멘토 종합 + 서명

한글 폰트(Noto Sans CJK)가 PDF에 임베드되어 어디서 열어도 깨지지 않습니다. 회차별 사진은 Notion 회차 페이지의 `📷 사진` 섹션에서 자동으로 가져오며, 도장 부분만 출력 후 수기로 처리하면 됩니다.

> **회차 일자**: 운영기관이 공지한 일자 기준으로 작성됩니다 (실제 진행일 아님). `⚙️ 멘토링 설정`의 "운영기관 공지 회차 일자" 섹션을 미리 채워두세요.

## Notion 회차 페이지 구조

`start-session`이 회차 페이지를 만들 때 아래 섹션 구조를 그대로 사용합니다. 모든 스킬이 이 섹션명을 기준으로 데이터를 읽고 씁니다.

| 섹션 | 용도 | 채우는 주체 |
|------|------|------------|
| `기본 정보` | 일시·장소·활동회차·활동주제 | start-session 자동 + 멘토 |
| `🔁 직전 회차 과제 점검` | 이전 과제 체크 표 | start-session 자동 |
| `🎯 이번 회차 어젠다` | 체크박스 어젠다 | start-session 자동 |
| `🗣️ 대화 노트` | 회차 진행 내용 | 멘토 직접 |
| `🤔 멘티가 한 말` | 멘티 발언 원문·인상적인 말·고민 | 멘토 직접 |
| `💡 멘토 코멘트` | 멘토가 준 조언·피드백 | 멘토 직접 또는 end-session |
| `📝 다음 회차 과제` | 체크박스 과제 목록 | 멘토 직접 또는 end-session |
| `📅 다음 멘토링 예정일` | 다음 회차 일자 | 멘토 직접 또는 end-session |
| `🤝 멘토 to-do` | 회차 사이 멘토가 할 일 | 멘토 직접 또는 end-session |
| `📨 회차 사이 업데이트` | 결과물 검토·미완 처리 기록 | between-sessions 자동 |
| `📷 사진` | 회차 사진 (최대 2장이 PDF에 삽입) | 멘토 직접 첨부 |

## 멘티 유형별 트랙

같은 "회사 추천"이라도 멘티 유형에 따라 다른 결과가 나옵니다.

- **구직자**: 합격 가능성 우선, 안전·메인·도전 분배
- **재직자-적응**: 회사 추천 보류, *"적응 진단부터"* 안내
- **재직자-이직**: 시장 가치 검증 위주
- **재직자-성장**: *"현재 회사 다음 챌린지"* 위주 제안
- **학생/취준**: 인턴·신입 풀, 안전 트랙 비중 높게

## 회차 유연성

기본은 3회차지만 자유롭게 조정할 수 있습니다.

- **연장 (4회+)**: 트래커에서 `계획 회차` +1 → 추가 회차 템플릿으로 자동 전환
- **축소 (1·2회)**: 시작할 때 `계획 회차`를 1 또는 2로 설정
- **조기 종료**: 진행 상태 `조기 종료`, 종료 사유 입력
- `create-report`가 회차 수와 종료 형태에 맞춰 보고서 톤·분량을 자동 조정

## 포함된 스킬

| 스킬 | 역할 | 시점 |
|------|------|------|
| `setup-mentor-toolkit` | 최초 셋업 | 처음 1회 |
| `init-mentoring` | 새 멘티 등록 + 분석 | 멘티 새로 받을 때 |
| `pre-assessment` | **참여자역량결과보고서(사전 평가지) PDF 생성** | **1차 시작 전** |
| `start-session` | N+1차 진입 — 컨텍스트 로드 + 브리핑 카드 + 회차 페이지 생성 | 2·3·4차 시작 직전 |
| **`end-session`** | **회차 직후 Notion 읽고 마무리 — 노트 구조화·트래커 갱신** | **각 회차 끝난 직후** |
| **`between-sessions`** | **회차 사이 결과물 검토·미완 과제 처리·진척 기록** | **회차와 회차 사이** |
| `mentee-analyzer` | 유형별 강점·약점·전략 분석 | init 자동 호출 + 재분석 시 |
| `company-recommender` | 분야·유형에 맞는 회사 추천 | "회사 추천해줘" 시점 |
| `resume-reviewer` | 이력서·포트폴리오 리뷰 | "이력서 봐줘" 시점 |
| `interview-prep` | 면접 질문 + 답변 가이드 | 면접 임박 |
| `create-report` | **운영기관 제출용 PDF 2종 생성 (멘토링 일지 + 결과보고서)** | 멘토링 종료 시 |
| `report-writer` | Notion 비공식 자유 정리 | 보조 |

## 포함된 Agents

스킬 내부에서 격리된 컨텍스트로 실행되는 전문 에이전트.

| 에이전트 | 역할 | 누가 호출하나 |
|---------|------|-------------|

## AI 티 안 나는 한국어

`create-report`는 보고서 작성 시 다음 규칙을 강제합니다.

- "다양한·지속적·체계적·효과적·전반적·향후·추후·사료된다" 같은 AI 시그널 단어 사용 금지
- 한국어 어미 다양화 (~다 / ~음 / ~한다 섞기)
- 구체적 회차·과제·사례 반드시 포함
- 강점·약점 분량 비대칭 허용
- 모든 멘티에 통하는 일반론 차단

각 텍스트 필드는 5개 자가 질문을 통과해야만 출력됩니다.

## FAQ

**Q. 다른 멘토에게 공유하면 그쪽도 똑같이 동작하나요?**  
네. 2단계 설치 후 `setup-mentor-toolkit` 한 번으로 본인 환경에 맞는 Notion 구조가 만들어집니다.

**Q. 분야를 바꾸고 싶을 때는?**  
`⚙️ 멘토링 설정` 페이지에서 직접 수정하면 다음 스킬 호출부터 반영됩니다.

**Q. 1회만 진행하고 보고서를 쓸 수 있나요?**  
네. `계획 회차`를 1로 설정하고 1차만 진행해도 `create-report`가 그에 맞는 보고서를 생성합니다.

**Q. 회차 사진은 어디에 넣나요?**  
Notion 회차 페이지의 `📷 사진` 섹션에 이미지를 첨부하면, `create-report` 실행 시 자동으로 가져와 PDF에 삽입합니다. 최대 2장.

**Q. PDF 저장 위치를 바꾸고 싶을 때는?**  
*"김지훈 보고서 PDF 만들어줘. ~/Documents/멘토링/에 저장해줘"* 처럼 경로를 함께 알려주면 됩니다.

## 개발자용 — 테스트

`build_pdfs.py`에 `--dry-run` 플래그가 있어 weasyprint·시스템 라이브러리 없이도 템플릿 회귀를 검증할 수 있습니다.

```bash
python3 -m venv ~/.venvs/mentor-toolkit
source ~/.venvs/mentor-toolkit/bin/activate
pip install jinja2 mplfonts pytest

pytest tests/ -v
```

자세한 내용은 [`tests/README.md`](tests/README.md) 참조.
