# mentor-toolkit

청년미래플러스 멘토링 자동화 **Claude Code** 플러그인. 멘토가 자기 환경(분야·소속·Notion)에 맞춰 1번 셋업하면, 그다음부터는 멘티 자료 경로만 알려줘도 멘토링 흐름이 굴러간다.

---

## 처음 사용하는 멘토 가이드

### 1단계. 사전 준비 (1회)

- ✅ **Claude Code CLI** 설치 — https://docs.claude.com/en/docs/claude-code
- ✅ **Notion MCP 연결** — `~/.claude.json`에 추가하거나 작업 디렉터리의 `.mcp.json`에 등록 (예: `claude mcp add notion ...`). Claude Code 공식 MCP 가이드 참고.
- ✅ **Python 3.10+ 의존성** (PDF 생성용) — 첫 보고서 생성 시 안내가 뜨지만 미리 설치하면 매끄러움:
  ```bash
  pip install --break-system-packages weasyprint jinja2 mplfonts requests
  ```
  macOS는 시스템 라이브러리가 더 필요할 수 있음:
  ```bash
  brew install pango libffi
  ```

### 2단계. 플러그인 설치

마켓플레이스 등록 후 플러그인 설치 — 두 단계:

```bash
# 1) 마켓플레이스 등록
claude plugin marketplace add https://github.com/vljh246v/mentor-toolkit

# 2) 플러그인 설치
claude plugin install mentor-toolkit@mentor-marketplace
```

설치 확인:

```bash
claude plugin list
```

업데이트 받기 (마켓플레이스 캐시 새로고침 후 재설치):

```bash
claude plugin marketplace update mentor-marketplace
claude plugin install mentor-toolkit@mentor-marketplace
```

제거:

```bash
claude plugin uninstall mentor-toolkit@mentor-marketplace
claude plugin marketplace remove mentor-marketplace
```

### 3단계. 셋업 (1회만)

Claude Code 세션을 열고:

```
> 셋업해줘
```

`setup-mentor-toolkit` 스킬이 발동되어 한 번에 묶어 묻는다:
1. 멘토 본인 성명
2. 소속
3. 직위
4. 멘토링 주 분야 (백엔드 / 프론트엔드 / 풀스택 / 모바일 / 데이터·AI / DevOps / UI/UX 디자인 / 서비스 기획·PM / 기타)
5. Notion에서 어디에 허브를 만들지 (자동 생성 / 기존 페이지 URL 제공)

답하면 자동으로 다음 생성:
- `📚 멘토링 자동화 허브` 페이지
- `⚙️ 멘토링 설정` (이후 분야·정보 변경은 여기서)
- `🧑‍💻 멘티 트래커` 데이터베이스
- `📝 템플릿 모음` 7장 (멘티 분석·1·2·3차·추가·조기 종료·결과 보고)
- `🚀 사용 가이드`

### 4단계. 첫 멘티 시작

이력서·자기소개서·포트폴리오 파일 경로를 알려주거나 멘티 이름·고민을 직접:

```
> 박다솔 멘토링 시작. 이력서: ~/Downloads/박다솔_이력서.pdf
```

또는

```
> 새 멘티 시작해줘
```

`init-mentoring`이 한 번에 묶어서 부족한 정보를 물어본 뒤 자동으로 등록·1차 분석·다음 액션 제안.

### 5단계. 회차 사이 — between-sessions가 다 처리

회차와 회차 사이에 일어나는 모든 것:

| 멘토가 한 말 | 처리 |
|------------|------|
| `박다솔 자소서 v2 봐줘 ~/Downloads/v2.txt` | resume-reviewer로 검토 → 점수·코멘트 + 멘티에게 보낼 답장 초안 + 1차 페이지에 자동 기록 |
| `박다솔 인프런 강의 못 들었대` | 안 한 이유 진단 + 4가지 옵션 (미루기/줄이기/대체/그만두기) + 추천 + 결정 반영 |
| `박다솔 강의 절반 들었대` | 부분 진척 기록 → 다음 회차 어젠다에 후반 마무리 추가 |
| `박다솔 1차 끝났어. 대충 이런 얘기...` | 자유 노트 → 1차 페이지의 대화/코멘트/과제 섹션에 자동 분배 |

다음 회차 시작 시 `start-session`이 이 모든 업데이트를 브리핑 카드에 자동 반영.

### 6단계. 다음 회차 시작 — 컨텍스트 자동 로드

1차가 끝난 지 2주 만에 2차 시작? 멘토 머릿속에서 멘티 컨텍스트가 흐릿해진 시점:

```
> 박다솔 2차 시작
```

`start-session` 스킬이 발동해서:
- 이전 회차 노트 + 멘티 분석 + 로드맵을 Notion에서 한 번에 끌어옴
- 새 회차 페이지 자동 생성 (직전 과제 점검 표 + 어젠다 미리 채워짐)
- 트래커 상태 갱신 (`1차 진행` → `2차 진행`)
- 멘토에게 **1분 브리핑 카드** 출력:
  - 멘티 한 줄 (유형·목표)
  - 직전 회차 핵심 + 부여한 과제
  - 누적 흐름 (1→N차)
  - 오늘 어젠다 4개
  - 시작 전 확인하면 좋은 질문 3개

이걸 슥 보고 미팅 시작하면 끝.

### 7단계. 멘토링 도중 자연어로 호출

| 멘토가 한 말 | 발동되는 스킬 |
|------------|------------|
| `박다솔 회사 추천해줘` | `company-recommender` |
| `박다솔 이력서 리뷰` | `resume-reviewer` |
| `박다솔 면접 질문 만들어줘` | `interview-prep` |
| `박다솔 모의 면접` | `mock-interviewer-agent` |
| `박다솔 분석 다시` | `mentee-analyzer` |

### 8단계. 멘토링 종료 — 공식 PDF 출력

```
> 박다솔 보고서 PDF 만들어줘
```

`create-report`가 **PDF 3개를 직접 생성**해서 현재 작업 디렉터리(또는 `--output-dir`로 지정한 경로)에 저장합니다:

- `박다솔_별지3-1_멘토링일지.pdf` (회차별 합본)
- `박다솔_별지3-2_결과보고서.pdf` (멘토 종합 + 서명)
- `박다솔_참여자역량결과보고서.pdf` (사전 평가지)

한글 폰트(Noto Sans CJK)가 PDF에 임베드되어 어디서 열어도 깨지지 않습니다. 회차별 사진은 Notion 멘티 페이지의 회차 하위 페이지에 멘토가 올려둔 이미지를 자동으로 가져옵니다. 도장(인) 부분만 출력 후 수기로 처리하면 끝.

**중요**: 회차 일자(`s1.datetime` 등)는 운영기관이 공지한 회차별 일자(예: 1회차 4/29 수)로 작성됩니다. 실제 멘토링 진행일이 아닙니다. `⚙️ 멘토링 설정` 페이지의 "운영기관 공지 회차 일자" 섹션을 갱신하세요.

## 분야별 자동 분기

설정에서 정한 주 분야에 따라 모든 스킬 결과가 다르게 나온다.

| 분야 | NCS 코드 (양식의 `mentoring.field`) | 평가 포커스 |
|------|---------------------------------|----------|
| 백엔드 / 프론트엔드 / 풀스택 / 모바일 / 데이터·AI / DevOps | `20. 정보기술` | 코드 깊이, 시스템 설계, 도메인 |
| UI/UX·시각·제품 디자인 | `08. 문화·예술·디자인·방송` | 포트폴리오·케이스 스터디·시각 완성도 |
| 서비스 기획·PM·PO·마케팅 | `02. 경영·회계·사무` | 문제 정의·KPI·문서화·실행력 |
| 기타 | 멘토가 직접 지정 | 일반 평가 + 분야 컨텍스트 |

설정 페이지에서 분야를 바꾸면 다음부터의 모든 스킬 출력이 그 분야 기준으로 바뀜.

## 멘티 유형별 트랙

같은 "회사 추천"이라도 멘티 유형에 따라 다른 답이 나간다.

- **구직자**: 합격 가능성 우선, 안전·메인·도전 분배
- **재직자-적응**: 회사 추천 보류, *"적응 진단부터"* 안내
- **재직자-이직**: 시장 가치 검증 위주
- **재직자-성장**: 추천보다 *"현재 회사 다음 챌린지"* 제안
- **학생/취준**: 인턴·신입 풀, 안전 트랙 비중 ↑

## 회차 유연성

기본 3회차이지만 자유롭게 조정 가능.

- **연장 (4회+)**: 트래커에서 `계획 회차` +1 → 추가 회차 템플릿 사용
- **축소 (1·2회)**: 시작할 때 `계획 회차`를 1·2로 설정
- **조기 종료**: 진행 상태 `조기 종료`, 종료 사유 입력
- `create-report`가 회차 수와 종료 형태에 맞춰 보고서 톤·분량 자동 조정

## 포함된 스킬

| 스킬 | 역할 | 시점 |
|------|------|------|
| `setup-mentor-toolkit` | 최초 셋업 | 처음 1회 |
| `init-mentoring` | 새 멘티 등록 + 1차 분석 | 멘티 새로 받을 때 |
| `start-session` | N+1차 진입 — 이전 회차 컨텍스트 자동 로드 + 브리핑 카드 | 2·3·4차 시작 직전 |
| **`between-sessions`** | **회차 사이 결과물 검토·미완 과제 처리·진척 기록·코멘트 초안** | **회차와 회차 사이 (가장 자주)** |
| `mentee-analyzer` | 유형별 강점·약점·전략 | init이 자동 호출 + 재분석 시 |
| `company-recommender` | 분야·유형에 맞는 회사 추천 | "회사 추천해줘" 시점 |
| `resume-reviewer` | 이력서·포트폴리오 리뷰 | "이력서 봐줘" 시점 |
| `interview-prep` | 면접 질문 + 답변 가이드 | 면접 임박 |
| `create-report` | **청년미래플러스 PDF 3종 직접 생성 (메인 결과물)** | 멘토링 종료 시 |
| `report-writer` | Notion 비공식 자유 정리 | 보조 |

## 포함된 Agents (서브에이전트)

스킬 내부에서 격리된 컨텍스트로 호출되는 전문 에이전트.

| 에이전트 | 역할 | 누가 호출하나 |
|---------|------|-------------|
| `company-research-agent` | 원티드/사람인/직행 병렬 fetch → 구조화된 JD 데이터 반환 | `company-recommender` 스킬 |
| `mock-interviewer-agent` | 격리된 채팅으로 모의 면접 진행 (질문→답변→피드백 사이클) | `interview-prep` 스킬 또는 멘토 직접 |

## AI 티 안 나는 한국어 보장

`create-report`는 보고서 작성 시 다음을 강제한다:
- "다양한·지속적·체계적·효과적·전반적·향후·추후·사료된다" 같은 AI 시그널 단어 추방
- 한국어 어미 다양화 (~다 / ~음 / ~한다 섞기)
- 구체적 회차·과제·코드 사례 ≥ 1개
- 강점·약점 분량 비대칭 허용
- 모든 멘티에게 통할 일반론 차단

각 텍스트 필드를 5개 자가 질문으로 검수해서 통과해야만 출력.

## FAQ

**Q. 다른 멘토에게 마켓플레이스를 공유하면 그쪽도 똑같이 동작하나요?**
A. 네. 그쪽 멘토도 위 2단계 (marketplace add + plugin install) 실행 후 `setup-mentor-toolkit`을 한 번만 호출하면 본인 환경에 맞는 Notion 구조와 설정이 자동으로 만들어집니다.

**Q. 분야를 바꾸고 싶을 때?**
A. Notion `⚙️ 멘토링 설정` 페이지에서 직접 수정. 다음 멘티부터 자동 반영.

**Q. 멘티 1명만 멘토링하고 보고서 쓸 수 있나요?**
A. 네. `계획 회차`를 1로 두고 1차만 진행해도 `create-report`가 그에 맞는 짧은 보고서를 생성합니다.

**Q. company-recommender의 회사 추천 데이터는?**
A. 백엔드 분야는 한국 시장 정적 데이터 + 원티드/사람인/직행 라이브 검색 둘 다 활용. 다른 분야는 라이브 검색 + AI 추론 위주.

**Q. PDF가 어디 저장되나요?**
A. 기본은 Claude Code의 현재 작업 디렉터리. 다른 곳에 저장하고 싶으면 *"박다솔 보고서 PDF 만들어줘. ~/Documents/멘토링/에 저장해줘"* 처럼 경로를 함께 알려주세요.
