# Init Mentoring — 멘토링 워크플로우 진입점

> **전제조건**: 시작 시 `references/preflight-check.md` 절차로 `⚙️ 멘토링 설정` 페이지 존재·무결성 검증. 부재/불완전이면 거기 명시된 표준 메시지 안내 후 종료.

새 멘티 1명을 받기 시작할 때 가장 먼저 발동되는 스킬. 시작 시 `⚙️ 멘토링 설정`을 읽어 멘토 분야를 확인. 이 스킬 하나로 다음을 처리한다.

1. 업로드된 문서(이력서/자소서/포트폴리오) 파싱
2. 멘티 유형 판별 (구직자 / 재직자-적응 / 재직자-이직 / 재직자-성장 / 학생/취준)
3. Notion 멘티 트래커 DB에 행 추가
4. Notion `🏡 멘토링` 허브 아래 멘티 이름 페이지 생성
5. 그 페이지 안에 "멘티 분석" 하위 페이지 생성 (분석 템플릿 적용)
6. `mentee-analyzer` 스킬을 자동 호출해 강점·약점·전략 채움
7. 다음 액션 3개 제안 (회사 추천 / 1차 멘토링 일정 / 사전 과제 부여)

## 입력 요건

다음 중 최소 1가지가 있어야 시작:
- 멘티 이름 + 자기소개 텍스트
- 이력서 파일 (PDF / docx / 이미지)
- 자기소개서 파일
- 포트폴리오 URL 또는 GitHub URL

부족하면 한 번에 묶어 묻기. 절대 1개씩 순차로 묻지 말 것.

## 처리 절차

### Step 1: 문서 파싱

사용자가 업로드한 파일을 읽는다. PDF면 pdf 스킬, docx면 docx 스킬을 사용. 추출 목표:

- 이름·나이·연락처
- 학력·전공
- 경력 (회사명·기간·역할)
- 기술 스택 (언어·프레임워크·DB·인프라)
- 자기소개에서 드러나는 가치관·고민·목표
- GitHub·포트폴리오 URL

### Step 2: 멘티 유형 자동 판별

추출 결과를 보고 다음 룰로 분류:

| 시그널 | 추정 유형 |
|--------|---------|
| 현재 무직 + 신입/이직 준비 단어 | 구직자 |
| 재직 중 + 입사 6개월 이내 + "적응" "혼자" 단어 | 재직자-적응 |
| 재직 중 + 이직 키워드("이직" "옮기고" "퇴사") | 재직자-이직 |
| 재직 중 + "성장" "시니어" "역량" 단어 + 안정적 재직 | 재직자-성장 |
| 학생·졸업예정·부트캠프 수료 직후 | 학생/취준 |

자신 없으면 멘토에게 한 번에 확인. 추측한 유형을 명시적으로 보여주고 "이 분류 맞나요? 다르면 알려주세요" 형태.

### Step 2.5: 동명 멘티 검사

트래커 DB를 조회해 같은 이름이 이미 등록돼 있으면 멘토에게 1회 묶어 묻기:
> *"`{이름}`이라는 이름이 이미 등록돼 있습니다 (시작일 YYYY-MM-DD, 유형 X). 새 멘토링 사이클을 시작하시는 건가요? 아니면 기존 멘티의 추가 회차인가요?"*

- "새 멘토링" → 트래커에 새 행 추가 (이름 뒤에 식별자, 예: "김지훈 (2)")
- "기존 멘티 추가 회차" → init-mentoring 종료, `start-session`으로 라우팅 안내

### Step 3: Notion 트래커 DB에 행 추가

`📚 멘토링 자동화 허브 > 🧑‍💻 멘티 트래커` DB에 새 페이지 생성. 채울 속성:

- 이름
- 분야 (자동: 백엔드, 다른 분야면 멘토 확인)
- 멘티 유형 (Step 2 결과)
- 시작일 (오늘)
- 진행 상태: `대기중` (분석 끝나면 `분석 완료`로 업데이트)
- **계획 회차**: 기본 3, 멘토가 미리 다른 숫자를 말하면 그대로 (예: 짧은 1회 단일 진단, 또는 6개월 6회)
- **진행한 회차**: 0
- GitHub (있으면)
- 현재 회사 (재직자만)
- 이직 희망 시기 (재직자-이직만)
- 주요 목표 (자소서에서 추출한 핵심 1~2줄)

**계획 회차 결정 로직**:
- 멘토가 명시 → 그대로
- 명시 없음 + 일반 멘티 → 3 (디폴트)
- 명시 없음 + 1회 진단 톤(*"한 번만 봐주세요"*) → 1
- 명시 없음 + 장기 톤(*"분기마다"*) → 멘토에게 한 번에 묶어 확인

회차는 언제든 늘리거나 줄일 수 있음을 멘티 트래커 메모에 표시.

### Step 4: Notion 멘티 페이지 생성

**`📚 멘토링 자동화 허브`** 안에 멘티 이름으로 빈 페이지 생성. 허브 페이지 ID는 `⚙️ 멘토링 설정` 페이지의 `parent.page_id`를 fetch해서 결정 (설정 페이지의 부모 = 허브). 페이지 ID를 받아 Step 3의 트래커 DB 행 `멘티 페이지` 컬럼에 URL을 입력.

### Step 5: 멘티 분석 페이지 생성

Step 4 페이지 안에 `멘티 분석` 하위 페이지 생성. 콘텐츠는 `📝 템플릿 모음 > 1️⃣ 멘티 분석 템플릿`을 복사한 구조. **입력** 섹션은 Step 1에서 추출한 정보로 미리 채움.

### Step 6: mentee-analyzer 호출

같은 플러그인의 `mentee-analyzer` 스킬을 호출해 **분석 결과** 섹션을 자동 작성. 강점·보완·방향성·유형별 전략·추천 학습 로드맵·리스크가 채워진다.

### Step 7: 트래커 상태 업데이트

분석 완료되면 트래커 DB 진행 상태를 `분석 완료`로 변경.

### Step 8: 다음 액션 3개 제안

멘티 유형에 따라 다음 3가지 중에서 가장 우선순위 높은 액션을 골라 제안:

| 유형 | 1순위 | 2순위 | 3순위 |
|------|------|------|------|
| 구직자 | 회사 추천 받기 | 이력서 리뷰 | 1차 멘토링 일정 |
| 재직자-적응 | 1차 멘토링 일정 | (회사 추천은 보류) | 사전 과제(1on1 요청) |
| 재직자-이직 | 회사 추천 받기 | 이력서 업데이트 리뷰 | 1차 멘토링 일정 |
| 재직자-성장 | 1차 멘토링 일정 | 학습 우선순위 정하기 | (이직 보류) |
| 학생/취준 | 포트폴리오 점검 | 인턴 채용 추천 | 1차 멘토링 일정 |

각 액션에 "이렇게 말씀하시면 됩니다" 예시 프롬프트 첨부:
- *"OO님 회사 추천해줘"* → company-recommender 발동
- *"OO님 이력서 리뷰해줘"* → resume-reviewer 발동
- *"OO님 1차 멘토링 시작"* → 1차 멘토링 페이지 생성 (별도 흐름)

## 출력 포맷

```
✅ 멘티 등록 완료: {이름}

📋 추출 정보
- 유형: {유형} ({근거 1줄})
- 경력: {연차}, {현 소속}
- 주 스택: {스택 요약}
- 핵심 목표: {목표 1줄}

📁 Notion에 생성됨
- 멘티 페이지: {URL}
- 멘티 분석: {URL}
- 멘티 트래커 행: {URL}

- 방향성: {1줄}

```

## 실패 케이스 처리

- **문서 못 읽음**: 멘토에게 텍스트로 붙여달라고 요청
- **유형 판별 애매**: 멘토에게 5개 유형 중 선택 요청 (한 번에 묶어)
- **Notion 권한 없음**: Notion MCP가 연결됐는지 확인하라는 안내
- **트래커 DB 못 찾음**: 멘토링 자동화 허브 페이지가 있는지 확인. 없으면 *"먼저 자동화 허브 셋업이 필요합니다"* 안내

## 회차 변경 시나리오 (등록 후 발생)

시작 후에도 회차가 바뀌는 케이스가 흔하다. init-mentoring은 등록만 담당하지만, 멘토가 다음 같은 말을 하면 트래커 DB만 업데이트:

- *"OO님 1회 더 추가하고 싶어"* → `계획 회차` +1, 진행 상태 `추가 회차 진행`으로 (기존 3차 끝났을 때)
- *"OO님 더 이상 진행 어렵게 됐어"* → 진행 상태 `조기 종료`, `종료 사유` 채움 → 멘티 페이지에 `🚫 조기 종료 메모 템플릿` 적용 → report-writer 호출 가능
- *"OO님 1회 단일 진단으로 마무리"* → `계획 회차`를 1로 조정

이 변경은 init-mentoring이 직접 처리하거나 멘토가 직접 트래커에서 수정해도 무방.

## 다른 스킬과의 관계

- **mentee-analyzer**: 이 스킬이 자동 호출. 단독으로도 호출 가능 (이미 등록된 멘티 재분석)
- **company-recommender**, **resume-reviewer**, **interview-prep**: 등록 후 멘토가 자연어로 호출
- **report-writer**: 멘토링 종료 시점에 호출. **진행한 회차가 1회든 5회든 동일하게 작동** (조기 종료 / 정상 완료 / 연장 완료 모두 처리)

## 멘토에게 한 번에 묶어 묻는 질문 예시

문서가 부족할 때 한 메시지에 모아서:

```
멘토링 등록을 시작하려면 다음을 알려주세요:
1. 멘티 이름:
2. 어떤 분이세요? (구직자 / 재직자-적응 / 재직자-이직 / 재직자-성장 / 학생/취준)
3. 다음 중 가지고 계신 자료를 올려주세요 (1개 이상):
   - 이력서 (PDF/docx)
   - 자기소개서 텍스트
   - GitHub URL
   - 포트폴리오 URL
4. 멘토링에서 가장 풀고 싶은 고민 1줄:
```

답이 오면 즉시 Step 3부터 진행.
