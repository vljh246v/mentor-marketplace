# 운영기관 제출 결과 문서 데이터 스키마

> `build_pdfs.py`가 입력으로 받는 평면 JSON의 키 명세. 키 명명은 정부 청년 멘토링 사업 운영 문서 양식의 필드 ID(별지 3-1·3-2 + 역량 결과보고서)를 그대로 따른다.
> create-report 스킬이 Notion에서 가져온 데이터를 이 스키마로 정리해 build_pdfs.py에 넘기면 PDF 3종이 직접 생성된다.

## 자동화 대상: 3종 6장

1. **멘토링 일지 [별지 3-1]** × 3회차 — 회차별 1장 = 3장
2. **멘토링 결과보고서 [별지 3-2]** — 멘토 종합 + 서명, 2장
3. **참여자 역량 결과보고서** — 사전 평가지, 1장 (담당 멘토가 담당 멘티 분량만 작성)

## 회차 진행일자 처리 (중요)

`s1.datetime` ~ `s3.datetime`은 **실제 멘토링 진행일이 아닐 수 있다.** 운영기관이 공지하는 회차별 일자(예: 1회차 = 4월 29일 수요일)에 맞춰 작성. 이 일자는 다음 중 하나에서 가져옴:
- `⚙️ 멘토링 설정` 페이지의 "운영기관 공지 회차 일자" 섹션
- 멘토가 호출 시 직접 입력 (*"1회차 4월 29일 수요일, 2회차 ...로 작성해줘"*)
- 위 둘 다 없으면 멘토에게 한 번에 묶어 질문

## 공통 필드

| 키 | 의미 | 출처 |
|----|------|------|
| `mentor.org` | 멘토 소속 | `⚙️ 멘토링 설정` 페이지 |
| `mentor.name` | 멘토 성명 | `⚙️ 멘토링 설정` 페이지 |
| `mentee.org` | 멘티 소속 | 멘티 분석 페이지 |
| `mentee.name` | 멘티 성명 | 멘티 페이지 제목 |
| `mentoring.field` | NCS 멘토링 분야 | `⚙️ 멘토링 설정` 페이지 (분야별 자동 매핑) |

### NCS 분야 코드 매핑

| 멘토링 분야 (설정에 입력하는 값) | `mentoring.field`에 들어갈 값 |
|---------------------------------|------------------------------|
| 백엔드 / 프론트엔드 / 풀스택 / 모바일 / 데이터·AI / DevOps | `20. 정보기술` |
| UI/UX·시각·제품 디자인 | `08. 문화·예술·디자인·방송` |
| 서비스 기획 / PM / PO / 마케팅 / 그로스 | `02. 경영·회계·사무` |
| 그 외 | 멘토가 설정 시 직접 NCS 코드 입력 |

## 멘토링 일지 (별지 3-1) — `s1.*`, `s2.*`, `s3.*`

| 키 | 의미 | 가이드 |
|----|------|--------|
| `sN.datetime` | 일시 | `2026.01.15.(목) 14:00~15:00 (1시간)` |
| `sN.place` | 장소 라디오 | `대면` 또는 `비대면` |
| `sN.placeDetail` | 장소 상세 | 카페명, Google Meet 등 |
| `sN.session` | 활동회차 | `구직청년 멘토링 N회차` 또는 `재직청년 멘토링 N회차` |
| `sN.topic` | 활동주제 | 한 줄 |
| `sN.content` | 멘토링 내용 (긴 텍스트) | ○ 4단 구조 |
| `sN.photo1`, `sN.photo2` | 사진 | Notion 회차 페이지의 📷 사진 섹션에서 URL 추출 → base64 인코딩 후 JSON에 포함. 사진 없으면 빈 문자열 `""`. 회차당 최대 2장, 초과 시 처음 2장만. |

### `sN.content` 작성 구조

```
○ 활동 도입 / 라포 형성
 - 멘티가 가져온 고민
 - (1·2회 제외) 직전 과제 점검

○ 핵심 활동
 - 다룬 주제 1
 - 다룬 주제 2

○ 멘토 조언 및 시사점
 - 구체적 조언

○ 다음 회차 사전 과제 (1·2회차)
 - 과제 1
 - 과제 2
```

3회차의 경우 마지막 항목을 "마무리·총괄"로 변경.
**재직청년 트랙 3회차**는 추가로 경력설계 로드맵 결과물 항목 명시:
```
○ 경력설계 로드맵 (재직청년 3회차 결과물)
 - 단기 (3개월): ...
 - 중기 (1년): ...
 - 장기 (3년): ...
```

## 멘토링 결과보고서 (별지 3-2)

| 키 | 의미 | 가이드 |
|----|------|--------|
| `r.outcome` | 멘토링 주요성과 | ○ 2~3개 항목 (목표 달성 + 정성적 변화) |
| `r.support` | 추가 지원 필요사항 | ○ 후속 지원 + 운영기관 제언 |
| `r.suggest` | 멘토링 건의 사항 | ○ 애로사항 + 프로그램 개선 제언 |
| `r.final` | 총평 (선택) | 멘티에게 남기는 한 단락 |
| `r.signYear` | 제출 연도 | `"2026"` |
| `r.signMonth` | 제출 월 | `"4"` |
| `r.signDay` | 제출 일 | `"30"` |
| `r.signOrg` | 멘토 소속명 | (mentor.org 동일) |
| `r.signTitle` | 직위 | `수석`, `책임`, `멘토` 등 |
| `r.signName` | 성명 | (mentor.name 동일) |

### `r.outcome` 형식
```
○ 초기 설정한 멘티의 목표와 기대사항 달성 정도
 - 시작 시 세운 목표
 - 달성도 + 구체적 결과물

○ 주요 성과 및 긍정적 변화
 - 진로/직무 방향 명확화
 - 구체적 변화 (포트폴리오, PR, 합격, 적응 등)
```

### `r.support` 형식
```
○ 멘토링 종료 후 멘티 지속 성장을 위해 필요한 추가 지원
 -

○ 운영기관의 추가 지원 필요사항
 -
```

### `r.suggest` 형식
```
○ 멘토링 수행 관련 애로사항
 -

○ 멘토링 프로그램 개선 제언
 -
```

## 참여자 역량 결과보고서 — `cap.*`

> 사전 평가지 성격. 운영기관 공지 일자 기준으로 작성한다 (실제 멘토링 진행일 아님).
> 담당 멘토가 담당 멘티 분량만 작성. 참여자 칸의 6명 멘토 그룹은 HTML 정적 영역.

| 키 | 의미 | 가이드 |
|----|------|--------|
| `cap.date` | 진행일자 | 운영기관 공지 일자 (예: `2026.04.29.(수)` — 1회차 기준) |
| `cap.targetJob` | 희망 직무 | 멘티 분석에서 가져옴 |
| `cap.docReview` | 제출 서류 검토 결과 | ● 4항목, 희망 직무 연관 평가 |
| `cap.completeness` | 강점·약점 요지 | ● 1줄 요지 + 서술 |
| `cap.plan` | 멘토링·역량 강화 방안 | ● 강점/약점에 1:1 대응 활동·강의 |

### `cap.docReview` 형식

운영기관 안내 톤에 맞춰 — *희망 직무와 연관지어 어떤 점이 좋고 부족한지* 간략하게.

```
● 제출 서류 : 이력서, 포트폴리오, 직무능력은행 계좌 인증서 (첨부 문서 기록)

● 이력서 평가
 : (희망 직무에 맞는 경험·지식 보유 여부, STAR·정량 지표 — 2~3문장)

● 포트폴리오 평가
 : (분야별 포커스: 백엔드/프론트는 기술 깊이·README, 디자인은 비주얼·케이스 스터디, 기획은 문서·KPI — 2~3문장)

● 직무능력은행 계좌 인증서 평가
 : (인증 항목 적절성 — 1~2문장)
```

### `cap.completeness` 형식

운영기관 예시 톤: *"개발 경험은 많은데 지식이 부족"* 같이 1줄 요지 + 서술.

```
(항목별로 나눠 강점 및 약점 작성)

● 강점 1 요지
 : 서술 (구체 사례·근거)

● 강점 2 요지 (필요 시)
 : 서술

● 약점 1 요지
 : 서술 (어떻게 부족한지)

● 약점 2 요지 (필요 시)
 : 서술
```

### `cap.plan` 형식

운영기관 예시: *"멘토링으로 부족한 지식 확인 후 해당 영역 집중 학습"*, *"인프런 강의 중 A기술 관련 강의 학습 제안"* 같은 톤.

```
(역량 및 서류 완성도를 기준으로 트랙에서 보완할 활동/강의 제안)

● 강점 1 요지 (cap.completeness 강점과 1:1)
 : 강점을 더 키울 활동·강의 (구체 명시)

● 약점 1 요지 (cap.completeness 약점과 1:1)
 : 약점을 메울 활동·강의 (예: 인프런 김영한 JPA 기본편, 사이드 프로젝트, 모의 면접)
```

## 트랙 구분

| 멘티 유형 | `sN.session` 값 |
|----------|----------------|
| 구직자 / 학생/취준 | `구직청년 멘토링 N회차` |
| 재직자-적응 / 재직자-이직 / 재직자-성장 | `재직청년 멘토링 N회차` |

## JSON 출력 형식

평면 JSON. 키는 점 표기 그대로.

```json
{
  "mentor.org": "...",
  "mentor.name": "...",
  "mentee.org": "...",
  "mentee.name": "...",
  "mentoring.field": "20. 정보기술",

  "s1.datetime": "...",
  "s1.place": "비대면",
  "s1.placeDetail": "Google Meet",
  "s1.session": "구직청년 멘토링 1회차",
  "s1.topic": "...",
  "s1.content": "...",

  "s2.*": "...",
  "s3.*": "...",

  "r.outcome": "...",
  "r.support": "...",
  "r.suggest": "...",
  "r.final": "...",
  "r.signYear": "2026",
  "r.signMonth": "4",
  "r.signDay": "30",
  "r.signOrg": "...",
  "r.signTitle": "...",
  "r.signName": "...",

  "cap.date": "...",
  "cap.targetJob": "...",
  "cap.docReview": "...",
  "cap.completeness": "...",
  "cap.plan": "..."
}
```

UTF-8, BOM 없음, LF 줄바꿈, 들여쓰기 2칸. 진행 안 한 회차는 빈 문자열로 두면 HTML이 빈 셀로 처리.
