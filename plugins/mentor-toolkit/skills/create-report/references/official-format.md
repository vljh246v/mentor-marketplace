# 운영기관 제출용 공식 결과보고서 양식

> 보고서 작성 시 **반드시 이 구조 그대로**. 섹션 추가·삭제·재배열 금지.

## 양식 사양

- 제목: **"참여자 역량 결과보고서"** (PDF 템플릿에서는 사업명 prefix가 헤더에 박혀 있음 — 양식 그대로 출력)
- 분량: 1페이지 (A4)
- 구조: 표(table) 1개. 행마다 섹션
- 폰트·서식: 워드 기본 + 표 테두리

## 4단계 표 구조

### 1. 헤더 (4행 × 2열)

| 항목 | 내용 |
|------|------|
| 진행일자 | YYYY.MM.DD.(요일) |
| 참여자 | (멘티명) |
| 희망 직무 | (멘티 분야 — 예: 백엔드 개발자, UI/UX 디자이너, 서비스 기획자 등) |
| 담당 멘토 | (담당 멘토 본인 — 설정에서 자동) |

**처리 규칙:**
- "(멘티명)" 자리에 실제 멘티 이름 + 괄호 표기 유지
- 담당 멘토는 `⚙️ 멘토링 설정` 페이지의 멘토 본인 정보 사용
- 희망 직무는 멘티 분석에서 추출한 직무로 채움

### 2. 제출 서류 검토 결과

```
● 제출 서류 : 이력서, 포트폴리오, 직무능력은행 계좌 인증서 (첨부 문서 기록)
● 이력서 평가
 : (한 단락 또는 2~3개 불릿)
● 포트폴리오 평가
 : (한 단락 또는 2~3개 불릿)
● 직무능력은행 계좌 인증서 평가
 : (한 줄 또는 두 줄)
```

각 평가는 *"제출되었음"* 같은 절차적 문장이 아니라 **내용 평가**여야 함:
- "이력서 평가": 구성·STAR·정량 지표·핏
- "포트폴리오 평가": 프로젝트 깊이·기술 스택·README 품질
- "직무능력은행 계좌 인증서 평가": 등록·인증 상태와 인증 항목의 적절성

### 3. 희망 직무 대비 서류 및 역량 완성도

```
● 강점 1 요지
 : (서술 — 구체적 근거)
● 강점 2 요지 (필요 시)
 : ...
● 약점 1 요지
 : (서술 — 어떻게 메울지 함께)
● 약점 2 요지 (필요 시)
 : ...
```

- **강점 2개·약점 2개 권장**. 부족하면 1개씩만.
- "요지"는 한 줄 헤드라인 (예: "백엔드 도메인 이해도 우수")
- ":" 다음에 **2~4문장** 서술. 너무 짧으면 인상 없고, 너무 길면 양식 분량 초과.

### 4. 멘토링 및 역량 강화 방안

```
● 강점 1 요지
 : (이 강점을 어떻게 더 키울지 — 활동·강의·과제 제안)
● 약점 1 요지
 : (이 약점을 어떻게 보완할지 — 활동·강의·과제 제안)
```

위 3번 섹션의 강점·약점과 1:1 대응. **각 항목에 구체적 활동·강의 1~2개 명시**.

활동 예시:
- 강의: 인프런 OOO 강의, 패스트캠퍼스 OOO, 학교 강의
- 활동: 사이드 프로젝트, 알고리즘 풀이, 모의 면접, 1on1, OSS 기여
- 과제: 이력서 v2 작성, 포트폴리오 README 정비, 시스템 설계 1문제 풀이

## 양식 외 정보 처리

다음은 **공식 보고서에 포함하지 않는다**:
- 멘토링 회차별 상세 노트 (Notion 회차 페이지 링크로 처리)
- 멘티 트래커 메타데이터
- 종료 사유 (조기 종료여도 양식상 표현하지 않거나, 약점 섹션에 자연스럽게 녹임)
- 멘티의 사적 정보 (개인사·건강 등)

## docx 출력 시 표 스타일

- 표 테두리: 단선
- 셀 padding: 보통 (워드 기본)
- 폰트: 맑은 고딕 10~11pt
- 헤더 행(섹션 제목)은 가운데 정렬·진하게
- 본문 행은 좌측 정렬

## 검수 체크리스트

작성 후 자체 검수:
- [ ] 진행일자 형식 YYYY.MM.DD.(요일)
- [ ] 4섹션 모두 채워짐
- [ ] 강점/약점 헤드라인 + 서술 형식 일관
- [ ] 강화 방안에 구체적 활동/강의 명시
- [ ] 한 페이지에 들어가는 분량
- [ ] 한글 깨짐 없음 (UTF-8)
