# Setup Mentor Toolkit — 최초 1회 셋업

플러그인 설치 후 멘토가 처음 사용할 때 **이 스킬을 가장 먼저** 발동해야 한다. 다른 스킬들은 이 스킬이 만든 Notion 구조와 설정에 의존한다.

## 만들어지는 것

다음을 자동 생성:

1. **`📚 멘토링 자동화 허브`** Notion 페이지 (사용자가 지정한 부모 페이지 아래)
2. **`⚙️ 멘토링 설정`** 페이지 — 멘토 본인 정보, 분야, NCS 코드 등 보관
3. **`🧑‍💻 멘티 트래커`** 데이터베이스 — 멘티 등록·진행 상태 관리
4. **`📝 템플릿 모음`** 페이지 + 7개 하위 템플릿:
   - 멘티 분석 템플릿
   - 1차 / 2차 / 3차 멘토링 템플릿
   - 추가 회차 템플릿 (4차+)
   - 조기 종료 메모 템플릿
   - 결과 보고서 템플릿 (Notion 비공식 보고용)
5. **`🚀 사용 가이드`** 페이지

## 처리 절차

### Step 1: 사전 점검

- Notion MCP가 연결돼 있는지 확인
- 안 돼 있으면 *"Notion MCP를 먼저 연결해주세요. Claude Code에서 `claude mcp add notion ...` 명령 또는 `~/.claude.json` / 작업 디렉터리 `.mcp.json`에 추가"* 안내 후 종료

### Step 2: 멘토 정보 한 번에 수집

다음을 한 메시지에 묶어 묻기. 한 개씩 순차 질문 금지.

```
mentor-toolkit 처음 사용이시네요. 셋업을 위해 다음만 알려주세요:

1. 멘토 본인 성명:
2. 소속 (예: 한국SW산업협회):
3. 직위 (예: 수석 / 책임 / 멘토):

4. 멘토링 주 분야 (가장 많이 다루는 1개):
   - 백엔드 / 프론트엔드 / 풀스택 / 모바일
   - 데이터·AI / DevOps
   - UI/UX·시각·제품 디자인
   - 서비스 기획 / PM / PO / 마케팅
   - 기타 (직접 입력)

5. Notion에서 어디에 허브를 만들까요?
   - 자동: "🏡 멘토링" 페이지를 새로 만들어 그 아래에 셋업
   - 수동: 사용자가 부모 페이지 URL 제공 (예: 기존에 멘토링 관련 페이지가 있다면 그 URL)
```

**부모 페이지 정책**: 부모 페이지의 ID는 `⚙️ 멘토링 설정` 페이지의 ancestor 메타로 자동 추적됩니다. 다른 스킬(init-mentoring·start-session 등)은 이 부모 페이지 아래에 멘티 페이지를 생성하므로, 한 번 설정한 부모는 일관되게 사용됩니다.

답이 부족하면 한 번 더 묶어 다시 묻기. 절대 1개씩 순차 질문하지 말 것.

### Step 3: NCS 분야 코드 자동 매핑

Step 2에서 받은 멘토링 주 분야를 NCS 코드로 변환:

| 입력 | NCS 코드 (`mentoring.field`) |
|------|------------------------------|
| 백엔드 / 프론트엔드 / 풀스택 / 모바일 / 데이터·AI / DevOps | `20. 정보기술` |
| UI/UX·시각·제품 디자인 | `08. 문화·예술·디자인·방송` |
| 서비스 기획 / PM / PO / 마케팅 / 그로스 | `02. 경영·회계·사무` |
| 기타 | 사용자가 NCS 코드 직접 입력하도록 한 번 더 묻기 |

### Step 4: Notion 부모 페이지 결정

- 사용자가 자동 선택 → Notion 워크스페이스 root에 새 `🏡 멘토링` 페이지 생성
- 사용자가 부모 페이지 URL 제공 → fetch해서 페이지 ID 추출

### Step 5: Notion 구조 일괄 생성

부모 페이지 아래에 다음 순서로 생성:

1. `멘토링 자동화 허브` 페이지 — icon: 📚
2. 그 아래 `멘티 트래커` 데이터베이스 — icon: 🧑‍💻 (스키마는 아래 참고)
3. 그 아래 `멘토링 설정` 페이지 — icon: ⚙️ — Step 2 답변을 바로 채움
4. 그 아래 `템플릿 모음` 페이지 — icon: 📝 + 7개 템플릿 하위 페이지
5. 그 아래 `사용 가이드` 페이지 — icon: 🚀

> **Notion 이모지 중복 방지**: 페이지 생성 시 이모지는 반드시 `icon` 필드(emoji 타입)에만 설정. `title` 속성에는 이모지 없이 텍스트만 입력. 이렇게 하지 않으면 Notion UI에서 `📚 📚 멘토링 자동화 허브`처럼 이모지가 두 번 보임.

각 페이지 ID를 다음 단계에서 사용하므로 보관.

### Step 6: 멘티 트래커 DB 스키마

### Step 7: 멘토링 설정 페이지 콘텐츠

다음 형식으로 채움:

```markdown

> mentor-toolkit 모든 스킬이 이 페이지를 참조합니다. 정보가 바뀌면 여기서 수정하세요.

## 멘토 본인 정보

- **성명**: {입력값}
- **소속**: {입력값}
- **직위**: {입력값}

## 멘토링 분야

- **주 분야**: {입력값}
- **NCS 분야 코드** (`mentoring.field`): {매핑된 코드}
- **세부 분야**: 백엔드 (예) — 멘티마다 다를 수 있음, 등록 시 트래커에서 지정

## 운영기관 공지 회차 일자

> 운영기관이 회차별로 공지하는 일자입니다. 실제 멘토링 진행일과 다를 수 있습니다.
> create-report가 양식의 `s1.datetime`, `s2.datetime`, `s3.datetime` 및 `cap.date`(= 1회차 일자) 채울 때 이 값을 사용합니다.
> 운영기관 공지가 새로 오면 여기서 갱신하세요. 형식은 `YYYY.MM.DD.(요일)` 권장 (예: `2026.04.29.(수)`).

- **1회차 공지 일자** (= 참여자 역량 결과보고서 진행일자): 
- **2회차 공지 일자**: 
- **3회차 공지 일자**: 

## 결과 산출물

create-report 스킬은 운영기관 제출용 PDF 3종을 직접 생성합니다 (외부 도구 없이 weasyprint로 한글 폰트 임베드):
- `{멘티명}_별지3-1_멘토링일지.pdf` — 진행한 회차 합본
- `{멘티명}_별지3-2_결과보고서.pdf` — 멘토 종합 + 서명
- `{멘티명}_참여자역량결과보고서.pdf` — 사전 평가지

## 트랙 구분 규칙

- **구직청년 멘토링 N회차** ← 멘티 유형이 구직자 / 학생·취준
- **재직청년 멘토링 N회차** ← 멘티 유형이 재직자-적응 / 재직자-이직 / 재직자-성장 (3회차 결과물로 경력설계 로드맵 필수)

## 회차 정책

- 기본 계획 회차: 3
- 멘티 상황에 따라 늘리거나 줄일 수 있음 (트래커에서 직접 수정)
- 조기 종료 시 종료 사유는 트래커 메모에 짧게, 보고서에는 직접 기재 안 함

---

*수정일: {오늘 날짜}*
```

### Step 8: 템플릿 모음 7장 생성

`references/templates/` 디렉터리의 마크다운 파일 7개를 **그대로** Notion 페이지로 복제. 본문을 즉석에서 만들지 말 것 (오타·구조 불일치 원인).

| 파일 | Notion 페이지 제목 | icon |
|------|------------------|------|
| `references/templates/01-mentee-analysis.md` | `멘티 분석 템플릿` | 1️⃣ |
| `references/templates/02-session-1.md` | `1차 멘토링 템플릿` | 2️⃣ |
| `references/templates/03-session-2.md` | `2차 멘토링 템플릿` | 3️⃣ |
| `references/templates/04-session-3.md` | `3차 멘토링 템플릿` | 4️⃣ |
| `references/templates/05-session-extra.md` | `추가 회차 템플릿 (4차+)` | ➕ |
| `references/templates/06-early-termination.md` | `조기 종료 메모 템플릿` | 🚫 |
| `references/templates/07-result-report.md` | `결과 보고서 템플릿 (Notion 비공식)` | 5️⃣ |

복제 절차:
1. 파일을 Read 도구로 읽음
2. 1번째 `# 제목` 라인은 페이지 title로 (이모지 제외)
3. 나머지 본문을 `notion-create-pages`의 content로 그대로 전달
4. `icon` 필드에 표의 이모지 설정 (title 속성에는 이모지 넣지 않음 — Step 5의 이모지 중복 방지 규칙 동일 적용)

분야별 특화는 스킬 실행 시 `⚙️ 멘토링 설정`을 읽어 동적으로 적용 (예: 재직청년 트랙은 3차에 경력설계 로드맵 섹션이 이미 템플릿에 포함됨).

### Step 9: 사용 가이드 페이지

다음 내용으로 작성:

```markdown
# 🚀 사용 가이드

## 새 멘티 시작
1. 멘티 자료 파일 경로를 알려주거나 *"새 멘티 시작"* 입력 (예: *"김지훈 멘토링 시작. 이력서: ~/Downloads/김지훈.pdf"*)
2. `init-mentoring` 스킬이 자동으로 등록·분석

## 멘토링 도중
- *"OO님 회사 추천해줘"* → company-recommender
- *"OO님 이력서 리뷰해줘"* → resume-reviewer
- *"OO님 면접 질문 만들어줘"* → interview-prep

## 멘토링 종료 시
*"OO님 보고서 PDF 만들어줘"* → create-report → 운영기관 제출용 PDF 3종 자동 생성
- 별지 3-1 멘토링 일지 / 별지 3-2 결과보고서 / 참여자 역량 결과보고서
- 한글 폰트 임베드, 사진은 Notion 회차 페이지에서 자동 fetch
- 도장(인) 부분만 출력 후 수기

## 회차 조정
- 연장: 트래커에서 `계획 회차` +1
- 조기 종료: 진행 상태 `조기 종료`로

## 분야 변경
`⚙️ 멘토링 설정` 페이지에서 주 분야·NCS 코드 수정 → 모든 스킬에 자동 반영
```

## 재실행 (이미 셋업된 경우)

이미 `📚 멘토링 자동화 허브`가 있으면:
- 사용자에게 *"이미 셋업되어 있습니다. 설정만 업데이트할까요? 아니면 처음부터 다시 만들까요?"* 묻기
- 설정만 업데이트: `⚙️ 멘토링 설정` 페이지만 갱신
- 처음부터: 기존 페이지를 백업 안내 후 새로 생성

## 에러 처리

- **Notion MCP 미연결**: 안내 후 종료
- **부모 페이지 권한 없음**: 다른 부모 페이지 URL 요청
- **DB 생성 실패**: 사용자에게 에러 메시지 그대로 보여주고 *"여기까지 만들어졌고 다음을 수동으로..."* 안내

## 다른 스킬과의 관계

- **이 스킬은 모든 스킬의 전제 조건**. 다른 스킬은 시작 시 `⚙️ 멘토링 설정` 페이지가 있는지 확인하고, 없으면 *"먼저 setup-mentor-toolkit을 실행해주세요"* 안내 후 종료해야 함.
- 이 스킬은 1회성. 분야 변경은 `⚙️ 멘토링 설정` 페이지 직접 수정으로 처리.
