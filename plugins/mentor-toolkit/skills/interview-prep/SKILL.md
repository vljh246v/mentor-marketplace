# Interview Prep

멘티 스택·경력·타겟 회사를 받아 맞춤 면접 질문 세트와 답변 가이드를 만든다.

## 모의 면접 모드 — `mock-interviewer-agent` 호출

멘토가 *"모의 면접 진행해줘"*라고 하면 question-set을 한 번에 출력하지 말고 격리된 면접관 에이전트를 호출:

```
Task tool 사용:
  subagent_type: "mock-interviewer-agent"
  prompt: 다음 5가지를 명시해서 전달
    - 타겟 회사
    - 포지션·연차
    - 면접 종류 (기술 / 시스템 설계 / 인성·경험 / 종합)
    - 세션 길이 (3문제 / 5문제 / 30분 풀코스)
    - 답변자 (멘티 본인 / 멘티 대신 멘토가 연습) — 명시 없으면 "멘토 연습"으로 기본
  → 에이전트가 질문 1개씩 던지며 답변 받고 피드백
  → 세션 종료 시 종합 리포트 반환
```

반면 *"면접 질문 5개 뽑아줘"* 같은 정적 요청은 이 스킬 본문이 직접 처리 (에이전트 호출 안 함).

## 위임 모드 (between-sessions에서 호출)

`between-sessions` 스킬이 멘티 자기소개 텍스트나 면접 답변 녹음 텍스트를 평가해달라고 위임 호출 시:
- 출력: 점수(★ 1~5) + 구체 코멘트 + 멘티에게 보낼 코멘트 초안
- between-sessions가 이 결과를 직전 회차 페이지의 "📨 회차 사이 업데이트" 섹션에 자동 기록

## 입력

필수:
- 멘티 정보 (스택·경력·유형) — Notion 페이지 URL 또는 직접
- 면접 종류 — 기술 / 시스템 설계 / 인성 / 모의 종합

선택 (있으면 정확도 ↑):
- 타겟 회사 (예: 토스, 우아한형제들)
- 단계 (서류 통과 후 1차 / 2차 / 임원)

## 질문 4개 카테고리

### 1. CS 기본기
멘티 연차에 따라 깊이 다름.

| 연차 | 깊이 |
|-----|------|
| 신입 | 자료구조, 알고리즘 시간복잡도, OS 프로세스/스레드, 네트워크 7계층, DB 정규화 등 교과서 수준 |
| 1~3년 | + 구체적 사례 ("실제 코드에서 락을 어떻게 잡았는지") |
| 3~7년 | 트레이드오프 토론, 깊은 OS·네트워크 |
| 7년+ | 시니어 — 케이스 스터디, 대규모 시스템 |

### 2. 언어/프레임워크 (멘티 주 스택)

Java/Spring 예시:
- JVM 메모리 구조, GC
- Spring Bean 생명주기, AOP
- JPA N+1, 영속성 컨텍스트, fetch 전략
- 트랜잭션 전파, Isolation
- Spring Security 흐름

Kotlin 추가:
- Coroutine vs Thread
- Null safety, sealed class

각 질문에 *"왜 묻는지"* 한 줄 설명 첨부 → 멘티가 답을 외우는 게 아니라 의미를 이해하도록.

### 3. 시스템 설계 (3년차+)

연차에 따라 난이도:
- 1~3년: 단일 도메인 설계 (예: 게시판, 채팅 메시지 저장)
- 3~7년: 여러 도메인 + 캐시/큐 (예: 결제 시스템, 알림 시스템)
- 7년+: 글로벌 스케일 (예: 트위터 타임라인, 카카오톡 메시지 전달)

각 설계 질문에 다음 4단계 답변 가이드:
1. 요구사항 명확화 (RPS, 지연, 일관성)
2. 큰 그림 컴포넌트 (LB, App, DB, Cache, Queue)
3. 데이터 모델·API
4. 병목·확장·장애 대응

### 4. 인성·경험 (모든 연차)

STAR 답변을 끌어낼 질문:
- "갈등 상황을 어떻게 풀었나"
- "실패한 프로젝트와 배운 점"
- "내가 한 가장 큰 결정"
- "팀에 가져다준 변화"

타겟 회사별 컬처 시그널:
- **토스**: 임팩트 / 솔직함 / 빠른 의사결정
- **우아한형제들**: 협업 / 사용자 중심 / 코드 품질
- **카카오**: 문제 정의 / 데이터 기반
- **네이버**: 도메인 깊이 / 안정성
- **쿠팡**: 영어·글로벌 / Customer obsession (Amazon DNA)
- **외국계**: 영어 인성, behavioral question 정형 (Tell me about a time...)

## 출력 포맷

### 모의 종합 모드 (full set)

```
## {이름} 면접 준비 ({타겟 회사 또는 일반})

### 자기소개 (90초 버전)
{멘티 정보 기반 모범 답안 1개}

### 1차: 기술 + CS (30분 가정, 12문제)

Q1. (CS 기본) {질문}
- 왜 묻는지: ...
- 답변 핵심 키워드: A, B, C
- 모범 답변 골격:
  ...
- 흔한 실수: ...
- 후속 질문: ...

Q2. (Java/Spring) ...
...

### 2차: 시스템 설계 (45분 가정, 1~2문제)

Q1. {설계 질문}
- 1단계: 요구사항 명확화 — {예시 질문 3개}
- 2단계: 큰 그림 — {권장 다이어그램 텍스트}
- 3단계: 데이터 모델 — {스키마 1개}
- 4단계: 병목·확장 — {3가지 시나리오}

### 3차: 인성·경험 (10문제)

Q1. {질문}
- 멘티가 쓸 만한 경험 후보 (Notion 페이지 기반): ...
- STAR 골격:
  S: ...
  T: ...
  A: ...
  R: ...

### 면접 직전 체크리스트
- [ ] 자기소개 90초 1번 녹음
- [ ] 시스템 설계 1문제 30분 모의
- [ ] 인성 답변 5개 STAR 구조로 적기
- [ ] 회사·제품 최근 뉴스 30분 리서치
- [ ] 역질문 3개 준비
```

### 빠른 모드 (한 카테고리만)
멘토가 *"기술 면접 질문 5개만"* 같이 요청하면 위 포맷 줄여서 5문제만.

## 멘티 유형별 톤

- **학생/취준 / 구직자 신입**: CS 기본 위주. 너무 어려운 시스템 설계는 1문제만.
- **구직자 경력**: 시스템 설계 비중 ↑. 인성·경험 STAR 정리 강화.
- **재직자-이직**: 현재 회사 사례를 어떻게 풀어 쓸지 코칭. 가장 임팩트 있는 부분.
- **재직자-성장**: 본인 회사 면접 패턴이 아니라 "내가 면접관으로 무엇을 묻는지"로 관점 전환.
- **재직자-적응**: 일반적으로 면접 준비 단계 아님. *"지금 면접 준비를 원하시는 이유"* 먼저 확인.

## 회사 정보 보강

타겟 회사가 명시되면:
- 그 회사 기술 블로그 1~2개 fetch (가능하면)
- 채용 공고 JD에서 자주 나오는 키워드 추출
- 그 회사가 자주 묻는다고 알려진 질문 패턴 반영

JD URL이 함께 주어지면 우선 fetch해서 그 JD 키워드에 정확히 맞춤.

## 모의 면접 진행

멘토가 *"모의 면접 진행해줘"*라고 하면 한 번에 모든 답을 보여주지 말고:
1. 질문 1개 던지고 멘토(또는 멘티)가 답할 시간 줌
2. 답이 들어오면 채점 + 피드백
3. 다음 질문

대화형 모드는 멘토 요청 시에만. 기본은 question-set 출력.

## 🪜 다음 권장 액션

> 출력 카드 마지막에 항상 1~3개 제시. 멘토가 그대로 복사해서 외칠 수 있는 트리거 문구 + 호출될 skill 이름 명시.

- *"OO님 모의 면접 진행해줘"* → interview-prep (mock-interviewer-agent 호출로 대화형 모의면접 진행)
- *"OO님 자소서 검토해줘"* → between-sessions (면접 준비와 병행한 회차 사이 결과물 관리)
