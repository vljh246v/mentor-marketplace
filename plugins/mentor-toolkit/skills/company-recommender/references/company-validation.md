# 회사 검증 게이트

추천 후보는 멘티 핏 점수화 전에 회사 검증을 먼저 통과해야 한다. 공고가 멘티에게 맞아도 회사 실체, 안정성, 주니어 적합성, 업무 리스크가 약하면 추천 등급을 낮추거나 제외한다.

## 필수 확인 항목

### 1. 공고 유효성
- 지원 가능 상태가 확인되는가.
- 마감일 또는 상시채용 여부가 확인되는가.
- 요구 경력, 담당 업무, 필요 기술이 구체적인가.
- JD 본문이 포괄적 문구만 반복하지 않는가.

### 2. 회사 실체
- 공식 홈페이지, 공식 채용 페이지, 제품/서비스 페이지 중 하나 이상이 확인되는가.
- 회사가 무엇으로 돈을 버는지 설명 가능한가.
- 사람인, 잡코리아, 원티드 회사 페이지 등에서 규모, 업력, 소재지, 사업 영역을 확인할 수 있는가.

### 3. 안정성
- 폐업, 임금체불, 대량 구조조정, 장기 채용 반복, 잦은 재공고 정황이 있는가.
- 최근 뉴스에서 중대한 부정 이슈가 반복 확인되는가.
- 너무 작은 초기 회사가 주니어에게 핵심 시스템을 단독으로 맡기는 구조인가.

### 4. 주니어 적합성
- 1~3년차가 배울 수 있는 팀 구조, 리뷰 체계, 리드/시니어 존재가 보이는가.
- JD가 "전체 구축", "단독 운영", "장애 전담", "즉시 실무 투입"에 치우쳐 있지 않은가.
- 기술 스택은 맞지만 교육/온보딩 없이 방치될 위험이 큰가.

### 5. 업무 리스크
- SI, SM, 파견, 상주, 운영대행 성격이 강한가.
- 야간, 주말, 장애 대응이 과도하게 요구될 가능성이 있는가.
- 멘티가 명시적으로 피한 도메인이나 업무 방식과 충돌하는가.

## 검증 소스 우선순위

1. 공식 홈페이지 또는 공식 채용 페이지
2. 채용 상세 JD 원문
3. 사람인, 잡코리아, 원티드 회사 페이지의 기업정보
4. 잡플래닛, 블라인드, 크레딧잡 등 평판 보조 소스
5. 최근 뉴스 검색
6. DART 또는 기업 공시가 있는 회사는 공시 정보

평판 사이트 점수는 단독 결정 근거로 쓰지 않는다. 동일한 부정 시그널이 2개 이상 출처에서 반복될 때 C 또는 D로 낮춘다.

## 검증 등급

- A: 공고, 회사 실체, 안정성, 주니어 적합성이 모두 양호하다. 메인 추천 가능.
- B: 일부 추가 확인은 필요하지만 큰 리스크가 없다. 추천 가능.
- C: 핏은 있으나 지역, 도메인, SI성, 평판, 온보딩 등 주의가 필요하다. 조건부 또는 안전 트랙만 가능.
- D: 공고 불명확, 회사 실체 부족, 주니어 방치 위험, 중대 평판 리스크가 있다. 추천 제외.

## 검증 신뢰도

- 높음: 상세 JD, 공식 홈페이지 또는 채용 페이지, 기업정보, 보조 평판/뉴스 중 3개 이상 확인. Scrapling MCP나 동등한 라이브 fetch로 상세 페이지를 검증함.
- 보통: 상세 JD와 기업정보는 확인했지만 공식/평판/뉴스 중 일부가 부족함. WebFetch/WebSearch fallback만 사용했을 수 있음.
- 낮음: 사용자 제공 JD 또는 검색 요약 중심이며 공식 회사 정보나 평판 확인이 부족함. 추천에는 반드시 제한 주석을 붙임.

## 즉시 제외 조건

- 지원 마감, 삭제, 비공개, 또는 지원 가능 상태 확인 불가
- 회사명, 서비스, 공식 정보가 확인되지 않음
- 임금체불, 폐업, 사기성 채용 정황
- 주니어에게 단독 구축, 단독 운영, 장애 대응 전체를 맡기는 형태
- 멘티가 명시적으로 피한 도메인이나 업무 방식과 정면 충돌

## 출력 규칙

각 추천 후보에는 다음을 포함한다.

- 검증 등급: A/B/C/D
- 검증 신뢰도: 높음/보통/낮음
- 수집 방식: Scrapling MCP / WebSearch fallback / 사용자 제공 JD
- 확인 출처: 확인한 JD, 공식 홈페이지, 기업정보, 평판/뉴스 소스
- 주요 리스크: 지역, 경력 갭, SI/SM/파견, 도메인, 온보딩, 평판 등
- 추천 판단: 지원 / 조건부 지원 / 보류 / 제외

검증 등급이 C 이하이면 점수보다 리스크 설명을 우선한다. 핏 점수가 높아도 D등급이면 추천하지 않는다.
