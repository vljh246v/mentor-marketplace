# 회사-멘티 매칭 룰

각 회사 후보에 5차원 점수(0~5) 매김. 합계 25점 만점.

## 1. 기술 스택 적합도

| 점수 | 기준 |
|-----|------|
| 5 | 회사 메인 스택과 멘티 보유 스택이 거의 일치 (Java/Spring + JPA + MySQL 둘 다) |
| 4 | 메인 스택의 80% 일치, 1개 정도 학습 필요 |
| 3 | 핵심 언어는 같지만 프레임워크·DB가 다름 |
| 2 | 언어부터 다름, 6개월+ 학습 필요 |
| 1 | 거의 처음부터 |
| 0 | 완전히 다른 스택 (예: Java만 한 사람이 Rust 회사) |

## 2. 경력 적합도

| 점수 | 기준 |
|-----|------|
| 5 | JD 요구 연차의 정확히 한가운데 (요구 3~5년에 4년차) |
| 4 | 요구 범위 안 |
| 3 | 약간 모자라거나 약간 넘침 (1년 이내 차이) |
| 2 | 2~3년 갭, 그러나 강점으로 메울 수 있음 |
| 1 | 갭이 크고 메우기 어려움 |
| 0 | 신입이 시니어 자리에 가는 식 |

## 3. 목표 적합도

멘티 유형별로 다르게 평가.

### 구직자 / 학생/취준
- 멘티가 "안정"을 원하면 → 대기업·중견 가산
- "성장 가속"을 원하면 → 시리즈 B~C 스타트업 가산
- "특정 도메인" 원하면 → 그 도메인 1~3위 회사 가산

### 재직자-이직
- 이직 사유가 "보상" → 외국계·쿠팡·토스 가산
- 이직 사유가 "도메인" → 그 도메인 회사 가산
- 이직 사유가 "조직 문화" → 컬처가 좋다고 알려진 곳 가산
- 이직 사유가 "기술 깊이" → 시니어 多 회사 가산

### 재직자-성장
- 현재 회사 시니어 풀이 부족 → 시니어 多 회사 가산
- 현재 회사 도메인 한계 → 다른 도메인 1위 가산

| 점수 | 기준 |
|-----|------|
| 5 | 멘티가 원하는 것을 그 회사가 정확히 제공 |
| 4 | 80% 매칭 |
| 3 | 일부 매칭, 일부는 안 맞음 |
| 2 | 한두 항목만 매칭 |
| 1 | 거의 안 맞음 |
| 0 | 멘티 우선순위와 정반대 |

## 4. 현실성 (통과 가능성)

| 점수 | 기준 |
|-----|------|
| 5 | 비슷한 프로필 합격 사례 다수, 코테·면접 난이도가 멘티 수준에 맞음 |
| 4 | 가능성 충분, 1~2개 부족점 보강 필요 |
| 3 | 도전 가능, 3개월 준비하면 합격선 |
| 2 | 6개월~1년 준비 필요 |
| 1 | 큰 갭, 다음 회사 거쳐서 가야 함 |
| 0 | 비현실적 |

체크 항목:
- 코딩테스트 난이도 vs 멘티 알고리즘 수준
- 시스템 설계 면접 비중 vs 멘티 설계 경험
- CS 깊이 vs 멘티 CS 베이스
- 컬처핏 면접 비중 vs 멘티 커뮤니케이션

## 5. 성장성

| 점수 | 기준 |
|-----|------|
| 5 | 합류 후 1년이면 멘티가 완전히 다른 사람이 됨 |
| 4 | 학습 곡선 가파름, 좋은 동료, 도메인 깊이 |
| 3 | 평균적인 성장, 안정적 |
| 2 | 학습 정체 가능성 |
| 1 | 장기 정체 위험 |

체크 항목:
- 시니어/스태프 비율
- 사내 발표·기술 블로그 활동성
- 도메인이 5년 후에도 의미 있는가
- 멘티가 그 회사에서 다음 회사로 자연스럽게 갈 수 있는가

## 가산점 룰

다음 시그널 있으면 +1~2점 (최대 25점 못 넘음):
- 멘티가 그 회사 출신 친구·선배 있음
- 멘티가 사용해본 서비스라 도메인 호감
- 영어 가능 + 외국계
- 멘티가 부트캠프 출신이면 부트캠프 친화 회사

## 감점 룰

다음 시그널 있으면 -1~3점:
- 멘티가 거주 못 하는 지역만 채용
- 멘티가 가지 않는 도메인 (예: SI 출신이 SI 회피하려 하는데 SI 회사)
- 워라밸 우선인데 격무 회사
- 멘티가 첫 회사인데 SI/SM만 있는 회사 (-3, 거의 비추)

## 출력 시 점수 표시 규칙

- 22~25점: ⭐⭐⭐⭐⭐ "최우선"
- 18~21점: ⭐⭐⭐⭐ "강력 추천"
- 14~17점: ⭐⭐⭐ "도전 가치 있음"
- 10~13점: ⭐⭐ "조건부 추천"
- 9점 이하: 추천 리스트에서 제외
