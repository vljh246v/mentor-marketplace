# 한국 회사 데이터베이스 (분야별)

> 멘토 시점 추정값. 정확한 처우·문화는 멘티가 직접 확인.
> 분야 무관하게 같은 회사가 여러 분야 채용을 진행하므로 일부 회사는 여러 섹션에 등장한다.

## 분야별 인덱스 (회사 추천 진입점)

- 백엔드 / 프론트엔드 / 풀스택 / 모바일 / 데이터·AI / DevOps → [개발 분야](#개발-분야)
- UI/UX·시각·제품 디자인 → [디자인 분야](#디자인-분야)
- 서비스 기획 / PM / PO / 마케팅 / 그로스 → [기획·PM 분야](#기획pm-분야)

## 개발 분야

## 분류 체계

회사를 4축으로 분류:
- **규모**: 대기업 / 중견 / 스타트업(시리즈 B+) / 초기 스타트업(A 이하·비공개)
- **도메인**: 커머스 / 핀테크 / 콘텐츠/플랫폼 / B2B SaaS / 게임 / 교육 / 모빌리티 / 헬스케어
- **기술 성향**: Java/Spring 보수 / Java/Spring 모던 / Kotlin / Node.js / Python / Go·Rust 도입형
- **합류 난이도**: ★(쉬움) ~ ★★★★★(빡셈)

## 주의 시그널 (피하거나 신중)

- **개발자 1~2명만 있는 회사** — 멘토 부재, 코드 품질 관리 어려움
- **SI/SM 위주 회사** — 도메인이 SI 자체가 아닌 한 학습 정체
- **시니어가 1년 안에 다 떠난 회사** — 조직 문화 위험 시그널
- **JD에 "주말·야간 가능자"가 명시된 회사** — 워라밸 위험
- **연봉 비공개 + 면접 끝나고도 안 알려주는 회사** — 보상 불투명

- **PM이 디자이너·개발자에게 산출물 발주만 하는 회사** — 진짜 의사결정 권한 없음
