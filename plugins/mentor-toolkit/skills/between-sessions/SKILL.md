# Between Sessions — 회차 사이 모든 작업 매니저

> **위치**: `init-mentoring`(등록) → `start-session`(회차 시작) **→ ⭐ between-sessions(회차 사이 작업)** → `start-session`(다음 회차) → `create-report`(종료)

회차와 회차 사이 1~3주 동안 일어나는 모든 일을 처리한다:

1. **결과물 검토** — 멘티가 보낸 자소서·이력서·PR·학습 노트
2. **미완 과제 처리** — 과제를 못 했을 때 미루기·줄이기·대체·그만두기 결정
3. **진척 기록** — 부분 완성을 직전 회차 페이지에 자동 기록
4. **멘토 답장 코멘트 초안** — 멘티에게 보낼 카톡/메일 한 단락
5. **다음 회차 어젠다 자동 조정** — 미완·재검토·후속 액션 반영

## 입력 패턴별 분기

호출 의도를 다음 4가지 중 하나로 식별:

### A. 결과물 검토 요청
- *"박다솔 자소서 v2 검토해줘"* + 자소서 첨부/텍스트
- *"박다솔 이력서 봐줘"* + 파일
- *"박다솔 PR 리뷰해줘 https://github.com/..."*
- *"박다솔 학습 노트 정리한 거 봐줄래"* + 텍스트

→ Step 1로 분기

> **트리거 라우팅 룰**: "OO 이력서 리뷰" 같은 발화는 `resume-reviewer` 단독으로도 후보가 될 수 있다. **회차 컨텍스트가 있을 때(트래커에 등록된 멘티 + 직전 회차 존재)는 between-sessions가 우선** — 검토 결과를 회차 페이지에 자동 기록하고 다음 회차 어젠다에 영향을 줘야 하기 때문. 미등록 멘티이거나 회차와 무관한 일반 리뷰면 `resume-reviewer` 단독 사용.

### B. 미완 과제 알림
- *"박다솔이 인프런 강의 못 들었대"*
- *"박다솔 자소서 아직 안 보냄"*
- *"박다솔이 이번 주 시간 없었대"*

→ Step 2로 분기

### C. 부분 진척 보고
- *"박다솔 강의 절반 들었대"*
- *"박다솔 PR 1개 머지했어"*
- *"박다솔 자소서 v1.5 정도 됐어"*

→ Step 3으로 분기

### D. 미팅 후 노트 정리 → end-session으로 라우팅
- *"박다솔 1차 끝났어"*, *"방금 1차 끝"*, *"N차 미팅 끝"*

→ **`end-session` 스킬 사용**. between-sessions가 아님.
  end-session은 Notion 회차 페이지를 먼저 읽고 비어있는 부분만 묻는다.
  멘토가 미팅 후 자유 노트를 Notion에 써뒀다면 질문 없이 자동 처리.

부족하면 한 번에 묶어서 짧게 묻기.

## Step 1: 결과물 검토

### 1-1. 멘티 식별 + 직전 회차 과제 매칭
- 트래커에서 멘티 식별
- 직전 회차의 "📝 다음 회차 과제" 섹션 fetch
- 들어온 결과물이 어느 과제와 매칭되는지 자동 판별

### 1-2. 결과물 종류별 검토 위임

| 결과물 | 호출 |
|--------|------|
| 이력서 / 경력기술서 / 자기소개서 | `resume-reviewer` 스킬 호출 |
| GitHub PR / 코드 | 메인 Claude가 직접 (필요 시 Read 도구로 코드 가져옴) |
| 포트폴리오 / 사이드 프로젝트 README | `resume-reviewer`의 포트폴리오 평가 섹션 |
| 학습 노트 / 정리 글 | 메인 Claude가 직접 (멘티 분야 기준 평가) |
| 자기소개 90초 녹음 텍스트 | `interview-prep`의 자기소개 평가 |

### 1-3. 검토 결과를 직전 회차 페이지에 기록

Notion 직전 회차 페이지의 `📨 회차 사이 업데이트` 섹션에 다음 형식으로 추가:

```
## {YYYY-MM-DD} 자기소개서 v2 검토

**대상 과제**: 자기소개서 v2 작성

**검토 점수**:
- STAR 구조: ★★★☆☆ (1차 대비 향상)
- 정량 지표: ★★☆☆☆ (여전히 부족)
- 도메인 핏: ★★★★☆

**핵심 코멘트**:
- 1차 대비 STAR 구조가 명확해짐
- 경력 2번에 정량 지표 1~2개 보강 필요
- ...
```

### 1-4. 멘티에게 보낼 답장 코멘트 초안

멘토가 카톡·이메일로 멘티에게 보낼 한 단락. 자연스러운 톤(natural-tone 규칙 적용):

```
"v2 잘 받았습니다. 1차 대비 STAR 구조가 많이 좋아졌네요.
다만 [경력 2번 항목]에 정량 지표(처리 건수·QPS 등) 1~2개만
들어가면 더 좋을 것 같습니다. 다음 회차에서 함께 마무리해보죠."
```

### 1-5. 다음 회차 어젠다 영향

검토 결과에 따라 다음 회차 페이지의 어젠다에 자동 추가:
- 결과물이 충분히 좋음 → 어젠다 변동 없음
- 보완 필요 → "OO 마무리 점검" 항목 추가
- 크게 부족 → "OO 처음부터 같이 다시" 항목 추가

## Step 2: 미완 과제 처리

### 2-1. 안 한 이유 진단

멘토 알림에서 원인 추출:
- 시간 부족 (회사 바쁨, 사적 사정) — 의지 문제 아님
- 어려움 (난이도 너무 높음, 시작점 모름) — 코칭 필요
- 의지 저하 (관심 떨어짐) — 동기 점검 필요
- 외부 변화 (이직 결정, 합격, 이사 등) — 우선순위 재조정

### 2-2. 4가지 옵션 제시 (트레이드오프 명시)

```
🔍 박다솔 인프런 강의 미완 진단
원인: 시간 부족 (회사 바쁨)

💡 옵션
1. 미루기 — 다음 회차까지 1주 더
   ✅ 학습 흐름 유지
   ⚠️ 또 못 할 가능성

2. 줄이기 — 전체 8시간 → 핵심 챕터 3시간만
   ✅ 현실성 높음
   ⚠️ 깊이 얕음

3. 대체 — 강의 → 책 1장 (1시간) 또는 짧은 영상 시리즈
   ✅ 시간 부담 ↓
   ⚠️ 강의 vs 책의 차이

4. 그만두기 — 이 과제는 의미 없음으로 결론
   ✅ 우선순위 정리
   ⚠️ 학습 갭 인정 필요

🎯 추천 (멘티 상황 기반)
2번 (줄이기). 회사가 단기 야근 시즌이라 시간 회복 후 전체 학습 가능.
지금은 N+1·인덱스 챕터 3시간만 들으면 다음 회차에 코드 리뷰 가능.

멘토님 결정 알려주세요.
```

### 2-3. 결정 반영

멘토가 옵션 선택하면:
- 직전 회차 페이지 `📨 회차 사이 업데이트`에 결정 + 사유 기록
- 다음 회차 페이지 어젠다에 반영 (예: 줄어든 과제 점검만)
- 트래커 메모에 짧게 기록 (조기 종료 시그널 누적되면 알림)

## Step 3: 부분 진척 보고

```
멘토: 박다솔 강의 절반 들었대. 자소서 v1.5 정도.

📝 박다솔 진척 기록
- 인프런 강의: 50% (4/8시간 완료)
- 자기소개서: v1.5 (v1보다는 낫지만 v2 미완)
- 미완 부분: 강의 4시간, 자기소개서 마무리

📁 1차 멘토링 페이지에 기록됨
🎯 2차 어젠다에 "강의 후반·자소서 마무리 함께 보기" 추가
```

## 작성 톤 (모든 출력 공통)

- AI 시그널 단어 추방 ("다양한·지속적·체계적·효과적")
- 어미 다양화
- 구체적 사례·시간·수치
- 멘티 결정 강요하지 않음 — 옵션 + 추천 형식

## 다른 스킬과의 관계

- **`end-session`**: "OO N차 끝" 발화는 end-session으로. 회차 직후 Notion 읽어 마무리 처리
- **`start-session`**: 다음 회차 시작 시 between-sessions가 남긴 업데이트·결정·미완 과제 자동 로드 → 브리핑 카드에 반영
- **`resume-reviewer`**: 결과물이 이력서·자소서면 이 스킬이 위임 호출
- **`interview-prep`**: 자기소개·답변 녹음 평가 시 위임 호출
- **`mentee-analyzer`**: 멘티 진척이 누적되어 이전 분석과 크게 다르면 *"분석 갱신할까요?"* 1회 제안

```
"..."
[복사해서 카톡·이메일 보내기]

🎯 다음 회차 영향: {1줄}
```

## 한국어·인코딩

- 모든 Notion 업데이트는 UTF-8
- 멘티 이름이 한글이면 부분 일치 검색 허용
- 결과물이 PDF/docx면 적절한 스킬·도구로 파싱 후 검토
