# Start Session — 회차 진입 브리핑

> **언제 발동**: 이미 등록된 멘티의 다음 회차를 시작할 때. 새 멘티 등록은 `init-mentoring`이 담당.

이 스킬은 회차 사이의 시간 간격을 **메우는** 역할이다. 멘토가 1차 끝낸 지 2주 만에 2차를 시작한다면, 멘토 머릿속에는 그동안 이 멘티 컨텍스트가 흐릿해져 있다. 이 스킬이:

1. 이전 회차 노트 + 분석 + 로드맵을 Notion에서 한 번에 끌어와 정리
2. 새 회차 페이지를 템플릿 + 직전 과제 자동 채움으로 미리 생성
3. 트래커 상태 갱신
4. 멘토에게 **회차 시작 직전 브리핑 카드** 전달 → 멘토는 그걸 슥 보고 미팅 시작

## 입력

다음 중 하나로 호출:
- 멘티 이름만 (*"김지훈 2차 시작"*) — 트래커에서 식별
- 멘티 페이지 URL — 직접 지정
- 멘티 이름 + 회차 명시 (*"홍길동 추가 회차"*) — 4차+

부족하면 한 번에 묶어 묻기. 절대 1개씩 순차 질문 금지.

## 처리 절차

### Step 1: 멘티 식별 + 회차 결정

1. **이름으로 트래커 검색** — 일치하는 멘티 1명 찾기
2. 다수 일치하면 멘토에게 1번에 묶어 후보 리스트 보여주고 선택받기
3. 트래커에서 다음 읽음:
   - 진행한 회차 N
   - 계획 회차 M
   - 진행 상태
   - 멘티 유형
   - 종료 사유 (있으면)

#### Step 1.5: 트래커 ↔ Notion 회차 페이지 자동 동기화 (회차 정합성 보정)

`end-session`을 멘토가 잊고 호출하지 않은 경우, 트래커의 `진행한 회차` N이 실제 진행 상태보다 뒤처질 수 있다. 이 단계에서 **회차 페이지 = 진실의 원천(source of truth)** 으로 보고 자동 보정한다.

1. 멘티 페이지 하위에서 회차 멘토링 페이지 개수 세기:
   - `1차 멘토링`, `2차 멘토링`, ..., `N차 멘토링` 형태 페이지 개수 = `M_pages`
2. 트래커의 N과 비교:
   - **N == M_pages** → 정상. 그대로 진행
   - **N < M_pages** → end-session 누락 가능성. 멘토에게 1회 확인:
     > *"트래커에는 진행한 회차가 N으로 되어 있는데, Notion에는 M_pages차까지 페이지가 있어요. 마지막으로 진행한 회차가 M_pages차 맞나요? 맞으면 트래커를 M_pages로 동기화하고 (M_pages+1)차를 시작합니다."*
   - **N > M_pages** → 데이터 불일치 (페이지 삭제 등). 멘토에게 알리고 수동 정리 안내
3. 멘토 확인 후 N을 M_pages로 갱신.

#### 다음 회차 번호 결정

동기화된 N 기준:
- 다음 회차 번호 = N + 1
- N + 1 ≤ M → 정상 진행
- N + 1 > M → "추가 회차 진행" 상태로 자동 전환 (또는 멘토에게 *"계획 M회를 넘기는 회차입니다. 연장하시겠어요?"* 1회 확인)
- 진행 상태가 `조기 종료` → "이미 종료된 멘토링입니다. 재개하시려면 `init-mentoring`으로 새 사이클 시작" 안내 후 종료

#### 멘토 발화의 회차 번호와 충돌 시

멘토가 *"OO 2차 시작"* 이라고 명시했는데 동기화 후 N+1 = 1차로 계산되면 충돌. 1회 묶어 묻기:
> *"멘토님은 2차 시작이라고 하셨는데, Notion에는 아직 1차 페이지만 있습니다. 어느 쪽이 맞나요? (1) 1차를 다시 정리 (2) 2차로 진행 (1차는 진행한 것으로 보고 동기화)"*

### Step 2: 모든 이전 컨텍스트 로드

병렬 fetch:
- 멘티 분석 페이지 → 강점·약점·로드맵·유형별 전략
- 1차 멘토링 페이지 (있을 때) → 다룬 주제·과제·코멘트·**📨 회차 사이 업데이트**·**🤝 멘토 to-do**
- 2차 멘토링 페이지 (있을 때) → 동일
- ... 직전 회차까지

각 회차에서 다음 추출:
- 다룬 주제 (1줄)
- 부여한 과제 → **이번 회차에서 점검할 항목**
- 멘토 코멘트 핵심
- 멘티의 반응·깨달음 시그널
- **📨 회차 사이 업데이트** 섹션 (between-sessions가 채운 내용):
  - 결과물 검토 결과 (자소서·PR·학습)
  - 미완 과제 처리 결정 (미루기/줄이기/대체/그만두기)
  - 부분 진척 기록
- **🤝 멘토 to-do** 섹션 — 멘토 본인이 회차 사이에 했어야 할 일

### Step 3: 오늘 어젠다 결정

다음 우선순위로 어젠다 후보 만듦:

1. **분석 단계 로드맵** — 멘티 분석 페이지의 "추천 학습 로드맵"에서 N+1차 부분
2. **직전 회차 과제 점검** — 자동으로 첫 어젠다
3. **유형별 회차 어젠다** — `mentee-analyzer`가 정한 유형별 트랙의 N+1차 핵심 (예: 구직자 2차는 "포트폴리오 리뷰 + 면접 대응")
4. **누적 시그널** — 1·2차 노트에서 "다음에 더 다뤄야겠다"는 흔적

이 4개를 합쳐서 3~5개 어젠다 항목으로 정리. 모순되면 멘토에게 *"이거 우선순위 어떠세요"* 1회 묻기.

### Step 4: 새 회차 페이지 생성

멘티 페이지 안에 `{N+1}차 멘토링` 하위 페이지 생성. 아래 섹션 구조를 **그대로** 사용한다.

> **규칙**: 섹션 제목에 괄호 주석 절대 금지. `(채움)`, `(자동)`, `(원문 근처)` 같은 메타 텍스트를 섹션명에 붙이지 않는다. 섹션 제목은 아래 명시된 이름 그대로.

#### 정규 회차 페이지 섹션 구조

```

- 일시: (빈 칸)
- 장소: (빈 칸)
- 활동회차: 구직청년 멘토링 {N+1}회차  ← 트랙 + 번호 자동 채움

(빈 칸 — 멘토가 준 조언·피드백 핵심)

4차+의 경우 `추가 회차 템플릿`을 사용. 조기 종료가 임박하면 (멘토가 "오늘이 마지막"이라고 알리면) 마무리 톤 어젠다.

### Step 5: 트래커 갱신

- 진행 상태: `{N+1}차 진행`으로 변경 (4차+면 `추가 회차 진행`)
- 최근 업데이트: 오늘
- 진행한 회차: N으로 유지 (회차 끝나면 N+1로 바꿈, 그건 별도 액션 또는 다음 호출 때)

### Step 6: 브리핑 카드 출력 (가장 중요)

멘토에게 **회차 시작 직전 1분 브리핑** 카드를 출력. 형식:

```
📋 {멘티 이름} {N+1}차 멘토링 브리핑

🧑 멘티 한 줄
- 유형: {구직자/재직자-적응/...} | 분야: {백엔드 etc.}
- 목표: {3회차에 얻고 싶은 것 — 분석에서}
- 트랙: {구직청년 / 재직청년}

- 다룬 주제: ...

이걸 본 멘토는 5분 내로 컨텍스트 복원하고 미팅 시작 가능.

## 회차 사이의 멘티 업데이트 처리

회차 사이에 멘티가 메일·메시지·Slack으로 진척을 알려오는 경우:
- 멘토가 *"OO님 1차 끝나고 어제 PR 머지했대"* 라고 알리면, 직전 회차 페이지의 **회차 사이 업데이트** 섹션에 추가 (없으면 만들기)
- 다음 회차 시작 시 그 정보가 자동으로 브리핑 카드에 반영됨

## 다른 스킬과의 관계

- **`init-mentoring`**: 새 멘티 첫 등록 (이 스킬과 짝)
- **`start-session` (이 스킬)**: 등록된 멘티의 N+1차 시작
- **`mentee-analyzer`**: 회차 사이에 *"OO 분석 다시"* 라고 하면 갱신
- **`create-report`**: 마지막 회차 끝난 뒤 PDF 보고서 생성

## 실패 케이스

- **멘티 못 찾음**: 트래커에서 일치 0 → *"등록된 멘티가 없습니다. `init-mentoring` 먼저 실행해주세요"*
- **분석 페이지 없음**: 트래커는 있는데 분석 안 된 상태 → *"분석을 먼저 진행할까요?"* 1회 묻기
- **이전 회차 페이지 비어 있음**: *"{N}차 노트가 비어 있어 점검할 과제가 없습니다. 어떤 어젠다로 진행하시겠어요?"* 1회 묻기

## 한국어·인코딩 주의

- 모든 Notion 페이지 fetch는 UTF-8
- 멘티 이름이 한글이면 트래커 검색 시 정확 일치 우선, 없으면 부분 일치
